# Day 1 — Azure Fundamentals: Core Concepts & Portal Deep Dive

> Phase 0 — Foundations + Account Setup

## Goal

Build a mental model of how Azure is organized before touching any services.

## Key Topics

- Azure Global Infrastructure: Regions, Availability Zones, Region Pairs
- Azure Resource Hierarchy:
  - Management Groups → Subscriptions → Resource Groups → Resources
- What is a Resource Group? A logical container that groups related Azure resources for management, billing, and access control
- Azure Resource Manager (ARM) — the control plane behind everything you do in Azure; every portal click, CLI command, and API call goes through ARM
- Naming conventions and tagging best practices
- Azure Marketplace overview
- Azure documentation and learning resources (Microsoft Learn)

## Hands-On Demo

**Account Requirements:** All steps are free tier — students can follow every step.

- ✅ Create a Resource Group
- ✅ Apply tags to it (e.g., Environment: Learning, Owner: YourName)
- ✅ Explore Azure Marketplace
- ✅ Browse Microsoft Learn (learn.microsoft.com)

## Summary

Azure organizes everything into a hierarchy: Management Groups → Subscriptions → Resource Groups → Resources. ARM is the engine that processes every action. Resource Groups are the everyday building block — everything you create lives inside one.
